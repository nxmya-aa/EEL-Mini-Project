How It Works

Step-by-step overview of sensing, processing and telemetry.

Hardware

  • Arduino Uno / Nano
  • Ultrasonic sensor (HC-SR04)
  • ESP8266 / ESP32 (Wi‑Fi telemetry)
  • Servo (optional) for automated lid
  • LED indicators / Buzzer

Software Flow

  1. Ultrasonic reads distance → Arduino converts to fill %
  2. Data packaged and sent via Wi-Fi (MQTT/HTTP) to server or cloud
  3. Dashboard receives telemetry and triggers alerts at thresholds
  4. Routing engine schedules pickups based on fill-priority

Calibration Tip

Calibrate distance mapping per-bin since bin heights vary. Use `map(distance, minDist, maxDist, 100, 0)` in Arduino to convert.

Arduino Pseudo Sketch

// Read ultrasonic and compute percentage
long distance = readUltrasonic();
int percent = map(distance, 2, 50, 100, 0); // adjust per bin
// Send via ESP (HTTP POST or MQTT)
